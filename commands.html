Contact
This command sends contact information to you in a DM.
.contact

@Echo#2101 how to contact?
Legal
This command sends legal information to you in a DM.
.legal

@Echo#2101 legal-agreement
Help
This command sends help to your DM on how to use Echo.
Optionally you can supply a command that will show more info about that command.
.help [Command]
Info
This command will show some statistics about Echo.
By using -extended Echo will also provide some technical data.
.info[-extended]
What is Echo’s prefix?
Forgot what Echo’s prefix was?
No problem! Ask Echo and he will tell you.
@Echo#2101 what is your prefix?
Enabling & Disabling commands.
With these commands you can enable/disable other specific commands.
You can also disable parts of a command.

(By default everything is enabled.)
.enable <Command>
.enableall

.disable <Command>

Example, this would disable the prefix command:
.disable prefix
Prefix
With this command you can change the prefix Echo will listen to.
Note that the prefix has a maximum length of 3 and the prefix “:” is reserved for the Intralink system.
.prefix <New Prefix>

Example, this would change the prefix to “!”:
.prefix !
Language
With this command you can change the language Echo will respond in.
You can use an abbreviation, the English name or the native name of the language.
.language <New Language>

Example, these would change the language to Dutch and French respectively:
.language NL
.language Français
Debug
With this command you can check the status of Echo’s permissions and the roles he can manage. Response will be send in DM.
.debug
Botmaster
With this command you can control who is recognised as a Master.

Who is recognised can rely on a role, a (combination of) permission(s) or a user list. You can change this by using the subcommand -mode.

If you switch to a certain mode for the first time Echo will automatically set some data for you.
For the user list mode this will be the owner ID, for the role mode the highest role in the server and for the permission mode the Manage Server permission.

If you want Echo to rely on a role or a (combination of) permission(s) you can change these respectively by using -role & -permission as subcommand. Permissions can either be provided as integer or by their names.

If you want to rely on a user list see the Add- & Delmaster section.
.botmaster-mode <Mode>
.botmaster-role <Role>
.botmaster-permission <Permission>

Example, this would change the system to rely on a role:
.botmaster-mode role

This would change the botmaster role to “Bot Commander” and change the required permission to “Administrator” & “Manage Server”:
.botmaster-role Bot Commander
.botmaster-permission Administrator, Manage Server
